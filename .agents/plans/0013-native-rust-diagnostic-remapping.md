# Native Rust diagnostic remapping

## Current gap

`RustSourceMap` maps generated item names to Jisp spans, but stores no byte or
line range in the rendered Rust file. Cargo/rustc JSON diagnostics identify a
generated file and byte/line column range, so item names alone cannot safely
map a compiler diagnostic back to Jisp.

## Required representation

Replace token-only emission at the CLI compilation boundary with a rendered
Rust artifact and an ordered map of generated byte ranges:

```text
GeneratedRustRange { generated_start, generated_end, source_span, kind }
```

At minimum, functions, generated structs, and enums get a range. Expression
range mapping can be added later without changing the JSON remapper API. A
diagnostic range maps to the smallest generated range containing its start;
otherwise it remains a generated-Rust diagnostic. Never guess an unrelated
Jisp span.

## CLI protocol

Add an explicit native-check/build command rather than changing `emit-rust`.
It writes a temporary Cargo crate, invokes Cargo with
`--message-format=json`, parses compiler messages, and renders remapped Jisp
diagnostics plus the original rustc message as a note. Generated files and
temporary paths are retained only under an opt-in debug flag. Proc macros keep
their existing limitation: stable spans may still point at macro invocation,
but the CLI path has precise generated-file coordinates.

## Tests

Unit-test range selection with synthetic JSON spans, including nested ranges,
out-of-range diagnostics, and imported Jisp source spans. Integration-test a
deliberately invalid generated Rust fixture only after the CLI owns a stable
temporary-crate protocol.
